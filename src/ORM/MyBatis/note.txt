1:读取mybatis配置文件到一个输入流
2:新建一个SqlSessionFactoryBuilder对象，该对象build()接收一个输入流，最终输出一个DefaultSqlSessionFactory对象(实现了SqlSessionFactory接口)
		1.build()方法内部将输入流转化为一个Configuration对象(bulid()方法有重载完整方法为:build(InputStream inputStream, String environment, Properties properties))
		2.然后根据Configuration对象作为参数创建一个DefaultSqlSessionFactory对象
3:DefaultSqlSessionFactory打开一个session
		1.默认使用openSessionFromDataSource(ExecutorType execType, TransactionIsolationLevel level, boolean autoCommit)
		      方法内部先获取Configuration对象的Environment属性对象
		      如果这个Environment对象为空或者对象内没有TransactionFactory接口类的对象，则新建一个ManagedTransactionFactory对象(实现了TransactionFactory接口)
		      否则返回Environment对象内的TransactionFactory接口类的对象(这个对象只可能是ManagedTransactionFactory对象或者JdbcTransactionFactory)
		     以上步骤必然会返回一个TransactionFactory接口类对象(一般用JdbcTransactionFactory对象)
		  利用获得到的TransactionFactory接口类对象调用newTransaction(environment.getDataSource(), level, autoCommit)获取Transaction对象

连接池配置参数说明：
poolMaximumActiveConnections = 10;    最大激活连接队列长度
poolMaximumIdleConnections = 5;       最大空闲连接队列长度
poolMaximumCheckoutTime = 20000;      当无法直接获取新连接（激活连接队列和空闲连接队列都满的情况下，如果激活连接队列内最老的连接check时间超过这个值就使用这个连接作为最新的连接）
poolTimeToWait = 20000;               当无法直接获取新连接（激活连接队列和空闲连接队列都满的情况下，如果激活连接队列内最老的连接check时间不超过poolMaximumCheckoutTime时需要方法wait()的时间）
poolPingQuery = "NO PING QUERY SET";  测试ping的sql
poolPingEnabled = false;              是否开启测试ping机制
poolPingConnectionsNotUsedFor = 0;    如果一个连接的getTimeElapsedSinceLastUse大于poolPingConnectionsNotUsedFor且ping机制开启则需要用sql测试ping
